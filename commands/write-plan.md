# Write Plan

Create a detailed implementation plan from an approved design document.

1. Read the most recent design document in docs/plans/.
2. Break it into bite-sized tasks (2-5 minutes each) with exact file paths, specific test cases, and verification steps.
3. Order tasks by dependency. Group independent tasks for parallel execution.
4. Tag each task with the appropriate agent tier (implementer-heavy or implementer-light).
5. Save the plan to docs/plans/YYYY-MM-DD-<feature-name>.md and commit.

For the full methodology, read ~/.code_puppy/skills/writing-plans/SKILL.md
